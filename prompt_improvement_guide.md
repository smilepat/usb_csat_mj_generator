## 1. 개선이 필요한 프롬프트 찾기

### 1.1 방법 1: 자동 개선 스캔 (가장 권장)

시스템이 자동으로 문제 있는 프롬프트를 찾아줍니다.

**실행 방법:**
1. **[프롬프트]** 메뉴 클릭
2. 상단의 **[🔍 자동 개선 스캔]** 버튼 클릭

```
┌─────────────────────────────────────────────────────────┐
│  🔍 자동 개선 스캔 결과                                  │
├─────────────────────────────────────────────────────────┤
│  스캔된 프롬프트: 45개                                   │
│  개선 필요: 5개                                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  🔴 심각 (즉시 개선 필요)                                │
│  ┌────────────────────────────────────────────────────┐ │
│  │ 키: 31                                             │ │
│  │ 제목: RC31 빈칸추론                                │ │
│  │ 문제: 승인율 42%, 연속 실패 5회                    │ │
│  │ [상세보기] [자동 개선]                             │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  🟡 주의 (개선 권장)                                     │
│  ┌────────────────────────────────────────────────────┐ │
│  │ 키: 29                                             │ │
│  │ 제목: RC29 어법                                    │ │
│  │ 문제: 평균 점수 65점, 승인율 58%                   │ │
│  │ [상세보기] [자동 개선]                             │ │
│  └────────────────────────────────────────────────────┘ │
│  ┌────────────────────────────────────────────────────┐ │
│  │ 키: 35                                             │ │
│  │ 제목: RC35 흐름무관                                │ │
│  │ 문제: 평균 점수 62점                               │ │
│  │ [상세보기] [자동 개선]                             │ │
│  └────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**자동 스캔 기준:**
| 상태 | 조건 | 의미 |
|------|------|------|
| 🔴 심각 | 연속 실패 3회 이상 | 즉시 수정 필요 |
| 🔴 심각 | 승인율 30% 미만 | 거의 작동하지 않음 |
| 🟡 주의 | 승인율 50% 미만 | 절반 이상 실패 |
| 🟡 주의 | 평균 점수 60점 미만 | 품질이 낮음 |
| 🟢 정상 | 위 조건 해당 없음 | 정상 작동 중 |

### 1.2 방법 2: 프롬프트 목록에서 직접 확인

**실행 방법:**
1. **[프롬프트]** 메뉴 클릭
2. "점수" 또는 "등급" 컬럼으로 정렬
3. C등급 이하 프롬프트 확인

```
┌─────────────────────────────────────────────────────────┐
│  📝 프롬프트 관리                    [점수 ▼ 오름차순]   │
├─────────────────────────────────────────────────────────┤
│  키      │ 제목           │ 점수 │ 등급 │ 승인율 │ 상태 │
│──────────────────────────────────────────────────────── │
│  31      │ RC31 빈칸      │ 52  │ F   │ 42%   │ ⚠️   │ ← 즉시 개선
│  35      │ RC35 흐름무관  │ 62  │ D   │ 55%   │ ⚠️   │ ← 개선 필요
│  29      │ RC29 어법      │ 68  │ D   │ 58%   │ ⚠️   │ ← 개선 필요
│  30      │ RC30 어휘      │ 75  │ C   │ 72%   │ -    │ ← 검토 권장
│  23      │ RC23 주제      │ 85  │ B   │ 88%   │ ✓    │
│  MASTER  │ 마스터 프롬프트 │ 92  │ A   │ 95%   │ ✓    │
└─────────────────────────────────────────────────────────┘
```

**우선순위 판단:**
- F등급 (60점 미만): **즉시** 개선
- D등급 (60-69점): **이번 주 내** 개선
- C등급 (70-79점): **여유 있을 때** 개선
- B등급 이상: 유지 (필요시 미세 조정)

### 1.3 방법 3: 생성 실패 후 역추적

문항 생성이 실패했을 때 해당 프롬프트를 확인합니다.

**실행 방법:**
1. **[요청목록]** 메뉴 클릭
2. 상태 필터: **FAIL** 선택
3. 실패한 문항의 프롬프트 확인

```
┌─────────────────────────────────────────────────────────┐
│  📋 요청 목록                        [상태: FAIL ▼]      │
├─────────────────────────────────────────────────────────┤
│  ID        │ 유형  │ 프롬프트 │ 상태  │ 실패 원인       │
│──────────────────────────────────────────────────────── │
│  abc-123   │ RC31  │ 31      │ FAIL │ JSON 파싱 실패  │
│  def-456   │ RC31  │ 31      │ FAIL │ 선택지 개수 부족│
│  ghi-789   │ RC31  │ 31      │ FAIL │ 정답 누락      │
└─────────────────────────────────────────────────────────┘

💡 같은 프롬프트(31번)에서 반복 실패 → 프롬프트 개선 필요!
```

### 2.3 Layer별 문제 파악

생성된 문항들의 품질 점수를 Layer별로 분석합니다.

| Layer | 점수 낮으면 | 프롬프트에서 확인할 것 |
|-------|-----------|---------------------|
| **Layer 1** (구조) | JSON 형식 오류, 필드 누락 | JSON 형식 예시 있는지, 필수 필드 명시했는지 |
| **Layer 2** (내용) | 정답 오류, 오답 매력도 낮음 | 정답 기준 명확한지, 오답 작성 가이드 있는지 |
| **Layer 3** (수능적합성) | 단어 수 부적절, 형식 불량 | 단어 수 범위 지정했는지, 수능 형식 명시했는지 |

## 3. 프롬프트 개선 방법

### 3.1 방법 1: AI 자동 개선 (가장 쉬움)

AI가 문제점을 분석하고 자동으로 개선안을 제시합니다.

**실행 방법:**
1. 해당 프롬프트 클릭
2. **[🤖 자동 개선]** 버튼 클릭
3. 개선 결과 확인 후 **[적용]** 또는 **[취소]**

```
┌─────────────────────────────────────────────────────────┐
│  🤖 자동 개선 실행                                       │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  분석 결과:                                              │
│  • 승인율: 42% (목표: 70% 이상)                         │
│  • 주요 실패 원인: JSON 형식 오류, 선택지 부족          │
│                                                          │
│  개선 내용 미리보기:                                     │
│  ┌────────────────────────────────────────────────────┐ │
│  │ [변경 1] JSON 형식 예시 추가                       │ │
│  │  + 반드시 다음 JSON 형식으로 출력하세요:           │ │
│  │  + { "passage": "...", "question": "...", ... }    │ │
│  │                                                    │ │
│  │ [변경 2] 선택지 개수 강조                          │ │
│  │  + ⚠️ 중요: 선택지는 반드시 5개를 생성하세요.      │ │
│  │                                                    │ │
│  │ [변경 3] 오답 작성 가이드 추가                     │ │
│  │  + 오답은 문맥상 그럴듯하지만 논리적으로 틀린      │ │
│  │    선택지로 작성하세요.                            │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  예상 효과:                                              │
│  • 승인율 42% → 70%+ 예상                               │
│  • JSON 형식 오류 감소                                  │
│                                                          │
│  [취소] [👁️ 전체 보기] [✅ 적용]                        │
└─────────────────────────────────────────────────────────┘
```

**[✅ 적용]** 클릭 시:
- 현재 버전이 히스토리에 백업됨
- 개선된 프롬프트가 적용됨
- 바로 문항 생성하여 효과 확인 가능

### 3.2 방법 2: 피드백 기반 개선 (맞춤형)

특정 문제에 대한 피드백을 입력하고 AI가 해결합니다.

**실행 방법:**
1. 해당 프롬프트 클릭
2. **[💡 개선 제안]** 버튼 클릭
3. 원하는 개선 사항 입력
4. **[🤖 AI 개선 실행]** 클릭

```
┌─────────────────────────────────────────────────────────┐
│  💡 피드백 기반 개선                                     │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  개선하고 싶은 내용을 자유롭게 입력하세요:               │
│                                                          │
│  ┌────────────────────────────────────────────────────┐ │
│  │ 1. 빈칸의 위치가 항상 문장 끝에만 있어요.          │ │
│  │    문장 중간이나 시작 부분에도 빈칸이 올 수 있게   │ │
│  │    해주세요.                                       │ │
│  │                                                    │ │
│  │ 2. 오답들이 너무 쉽게 배제됩니다. 정답과 비슷한   │ │
│  │    의미를 가진 오답을 만들어주세요.                │ │
│  │                                                    │ │
│  │ 3. 지문 주제가 너무 단순해요. 좀 더 학술적인      │ │
│  │    주제를 다뤄주세요.                              │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  [취소] [🤖 AI 개선 실행]                                │
└─────────────────────────────────────────────────────────┘
```

**[🤖 AI 개선 실행]** 클릭 후:

```
┌─────────────────────────────────────────────────────────┐
│  ✅ 개선 완료                                            │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  적용된 변경 사항:                                       │
│                                                          │
│  1. ✅ 빈칸 위치 다양화 지침 추가                        │
│     "빈칸은 문장의 시작, 중간, 끝 위치에 다양하게       │
│      배치하세요. 특히 핵심 논지가 담긴 위치를 선택..."   │
│                                                          │
│  2. ✅ 오답 매력도 향상 지침 추가                        │
│     "오답은 정답과 의미적으로 유사하거나, 지문의 다른   │
│      부분에서 언급된 개념을 활용하여 매력적으로..."      │
│                                                          │
│  3. ✅ 주제 다양화 지침 추가                             │
│     "인문, 사회, 과학, 예술, 기술 등 학술적 주제를      │
│      다루되, 수능 수준에 적합한 난이도로..."             │
│                                                          │
│  [이전 버전 보기] [✅ 확인]                              │
└─────────────────────────────────────────────────────────┘
```

### 3.3 방법 3: 수동 편집 (완전한 제어)

직접 프롬프트를 수정합니다. 정확히 무엇을 바꿔야 하는지 알 때 사용합니다.

**실행 방법:**
1. 해당 프롬프트 클릭
2. **[✏️ 편집]** 버튼 클릭
3. 직접 수정
4. 변경 사유 입력
5. **[💾 저장]** 클릭

**수정 시 체크리스트:**

```
┌─────────────────────────────────────────────────────────┐
│  ✏️ 프롬프트 편집                                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  수정 전 확인사항:                                       │
│  ☑️ JSON 형식 예시가 있는가?                             │
│  ☑️ 필수 필드(passage, question, options, answer)       │
│     명시되어 있는가?                                     │

## 5. 실전 예제: 29번 어법 프롬프트 개선

실제로 29번(어법) 프롬프트를 개선하는 전체 과정입니다.

### Step 1: 문제 발견

자동 스캔 결과에서 29번 프롬프트가 "주의" 상태로 표시됨.

```
🟡 29번 프롬프트
   승인율: 58% (목표: 70%)
   평균 점수: 65점
```

### Step 2: 문제 진단

**[📊 성능 분석]** 결과:
```
실패 원인:
1. 밑줄 친 부분 개수 불일치 (5개 아님): 35%
2. 정답이 틀리지 않은 어법 사용: 25%
3. 해설 누락: 20%
4. 기타: 20%
```

**[🤖 AI 평가]** 결과:
```
점수: 6.5/10
문제점:
- 밑줄 친 5개 부분 강조 부족
- 어법 오류 유형 예시 없음
- 정답의 오류가 왜 오류인지 설명 지침 없음
```

### Step 3: 개선 실행

**[💡 개선 제안]**에 입력:
```
1. 밑줄 친 부분이 정확히 5개가 되도록 강조해주세요
2. 정답은 반드시 명확한 어법 오류여야 합니다
3. 어법 오류 유형 예시를 추가해주세요 (관계사, 시제, 수일치 등)
4. 해설에서 왜 틀린지 명확히 설명하도록 해주세요
```

**[🤖 AI 개선 실행]** 클릭

### Step 4: 개선 결과 확인

```
┌─────────────────────────────────────────────────────────┐
